Para o provisionamento da infraestrutura do Crafty Controller, utilizei a nuvem da Amazon Web Services (AWS), onde criei uma instância EC2 (Elastic Compute Cloud) dedicada, nomeada como crafty. O sistema operacional selecionado foi o Ubuntu Server 26.04 LTS (64-bit). Como especificação de hardware para suportar a aplicação e os servidores de jogos com estabilidade, adotei uma instância do tipo m7i-flex.large, equipada com 2 vCPUs e 8 GB de memória RAM. Para assegurar o acesso remoto à máquina, a autenticação foi estruturada por meio de um par de chaves criptográficas utilizando o algoritmo de alta segurança ED25519, com a chave privada exportada e salva no formato .ppk para conexões via SSH.

Na camada de segurança periférica da nuvem, configurei um Security Group para gerenciar de forma estrita o controle de tráfego de entrada (Inbound Rules). Foram liberadas as portas padrão para comunicações via SSH (22), HTTP (80) e HTTPS (443). Adicionalmente, foi necessário implementar uma regra de TCP Personalizado na porta 8443, que corresponde à porta nativa utilizada pelo painel web do Crafty Controller. Essa regra foi definida com o range de origem global 0.0.0.0/0, o que viabiliza o acesso externo à interface gráfica de gerenciamento a partir de qualquer endereço IP na internet, eliminando restrições geográficas de acesso para os administradores.

Após a consolidação do ambiente de rede, realizei a instalação do ecossistema seguindo os scripts e diretrizes contidos na documentação oficial do Crafty Controller. Visando adotar as melhores práticas de administração de sistemas (SysAdmin), apliquei um refinamento na pós-instalação: a aplicação foi isolada dentro de um ambiente virtual de Python (.venv) e configurada como um daemon nativo do Linux via systemd. Essa abordagem garantiu que o painel passasse a rodar de forma resiliente em segundo plano (background) sob um usuário de menores privilégios (crafty), contando com políticas de autorrecuperação (on-failure) e inicialização automatizada diretamente no boot da instância AWS, eliminando a dependência de sessões ativas de terminal.
